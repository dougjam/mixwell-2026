A **Reverse-Drift Function** (RDF) is a displacement field that encodes how a
cylindrical tine inserts into, moves through, and withdraws from a 2D fluid.
Given an output coordinate, an RDF answers a single question: *where did this
material come from?* Because the answer is evaluated independently per sample,
RDFs behave a lot like signed distance fields — they **compose**, and can be
**chained** in a shader without ever resampling an intermediate texture.

## One equation

For a tine of radius <Math tex="a" /> centered at the origin, the idealized
potential-flow displacement at a point <Math tex="\mathbf{x}" /> (with
<Math tex="r = \lVert \mathbf{x} \rVert" />) can be written schematically as:

<Math
  display
  tex={String.raw`\mathbf{d}(\mathbf{x}) = \left(1 - \frac{a^2}{r^2}\right)\mathbf{x} + \frac{a^2}{r^2}\big(2(\hat{\mathbf{u}}\cdot\hat{\mathbf{x}})\,\hat{\mathbf{x}} - \hat{\mathbf{u}}\big)s`}
/>

  float t = iTime * 0.35;
  // Swirl around a moving "tine" center.

}

## Chaining RDFs

RDFs compose by function composition — feed the output coordinate of one into the
next, then sample the source once at the end:

<CodeBlock
  lang="glsl"
  code={`// Chain two RDFs, then sample the source exactly once (no intermediate blur).
vec2 uv = fragUV;
uv = rdfTine(uv, tineA, radiusA, driftA);
uv = rdfTine(uv, tineB, radiusB, driftB);
vec3 color = texture(uSource, uv).rgb;`}
/>
